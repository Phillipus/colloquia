21st March 2000 - Version 1.2 beta

Colloquia version 1.2 has undergone many changes since version 1.1 (formerly known as Learning Landscapes). Some of these changes are documented here and will mainly be of interest to users who have been using version 1.1 and are now upgrading to version 1.2.

  • Multi-user Log-on
    Version 1.1 saved only one set of user settings (stored in a file called ll.prefs). This settings file pointed to one data location. Version 1.2 now allows more than one user to log on by saving separate settings files for each user (in the format user_name.cqp) in a Colloquia folder. By default, this settings file points to a separate data folder (in the format user_name.cqd). It is recommended that users use these default locations.

A version 1.2 installation will look for an old version 1.1 ll.prefs file and rename it to user_name.cqp (or current user.cqp). THIS IS YOUR EXISTING PROFILE and points to your original data folder location. You do not need to add a new user profile.

Users may wish to rename their profiles from the LogOn window. This will rename the cqp file, but not the data folder location.
 

  • XML/HTML Format
    All data and text (tree data, notes, messages) is now saved in XML/HTML format. Version 1.1 users can convert their existing data to 1.2 format by selecting the Convert 1.1 Files option in the File menu from within Colloquia. Version 1.1 tree data is converted automatically to 1.2 format. Note - there is now only ONE data tree file per user - this is named data.tml and is stored in the tree folder.
     
  • E-mail composer
    The Colloquia e-mail composer now supports basic HTML formatting and the inclusion of any number of attachments.
     
  • Tree Folder Structure / Tree View
    The Tree Folder structure has been much simplified so that People, Resources, Assignment and sub-Activities are placed directly into an Activity Folder. Also, the Future, Live and Completed folders have been removed. At the top of the tree panel is a set of buttons that allow the user to turn off/on the tree display of People, Resources, Assignments, Completed Activities and Future Activities.
     
  • Templates
    Users can now drag existing Activities into the Templates folder to create a skeletal Activity structure that can act as the basis of a new Activity.
