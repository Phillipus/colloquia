Appendix 8: Changing the language and wording of Colloquia

Colloquia has multi-language support and you may wish to define your own language set or edit some words or phrases to suit your environment. Here's the background:

The main Java files that Colloquia uses are stored in a file named colloquia.jar found in the Colloquia install folder. This file is essentially an archive file that can be opened as if it were a zip file (PC users can use WinZip to open the file.) Amongst the many files contained in this is a file called strings.properties. If you wish to create new strings (text lines) based on this file, extract a copy of it from the colloquia.jar file (do not remove the file or Colloquia will no longer work) and open it in a text editor.

The first few lines of this file look something like this:

# English / Default

# General re-usable Strings
# ------------------------------------------------------------------------------
CANCEL=Cancel
OK=OK
ME=Me
YES=Yes
NO=No

The hash sign (#) indicates a comment line and any text to the right of this is ignored.

The word in capitals to the left of the equals sign (=) is the key and should not be changed.

The text to the right of the equals sign is the text that is displayed in Colloquia in menus, dialog boxes, information screens and so on.

You may create an edited version of this strings.properties file renamed to stringsX.properties (where X is a number) in the Colloquia install folder.

For example:

1. Using a zip utility such as WinZip, extract a copy of the strings.properties file from the colloquia.jar file into the Colloquia install folder (i.e. the same folder as that where colloquia.jar resides). Rename this extracted file to strings1.properties.

2. Open the strings1.properties file in a text editor and edit the words to the right of the equals sign.

3. Save the file.

When Colloquia next launches and finds this new file it will appear as a language option on the logon screen.

Further language sets may be created in the same way. Ensure that the named numbering sequence is preserved (strings1.properties, strings2.properties, strings3.properties and so on for each set.
